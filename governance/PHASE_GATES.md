# CPIMS Phase Gates
## Entry and Exit Control for Every Phase

This document defines the entry and exit gates for CPIMS phases.

No phase may begin or close without satisfying these conditions.

---

## 1. Gate Principles

### Entry Gate
A phase may begin only when:
- prerequisite truth is stable
- impacted metrics are defined
- state implications are understood
- seeded scenarios support the work
- public/internal boundaries are clear

### Exit Gate
A phase may close only when:
- feature behavior matches truth
- citizen, staff, and dashboard views are aligned
- required artifacts are reviewed
- browser walkthrough passes
- known limitations are recorded

---

## 2. Phase A — Truth Foundation

### Entry Gate
- mission approved
- governance direction approved

### Exit Gate
- truth registry exists
- lifecycle truth defined
- metric baseline defined
- governance model operational

---

## 3. Phase B — Core Experience Build

### Entry Gate
- schedule truth stable
- lifecycle truth stable
- seeded records available
- citizen copy principles defined

### Exit Gate
- citizen can understand today’s rule
- citizen can follow the correct path
- request submission works where required
- tracking works with public-safe visibility
- browser walkthrough proves the golden path

---

## 4. Phase C — Staff and Dashboard

### Entry Gate
- citizen flow stable
- lifecycle transitions approved
- dashboard metric definitions approved

### Exit Gate
- staff actions work correctly
- citizen tracker reflects staff actions appropriately
- dashboard reflects same record truth
- internal notes do not leak publicly
- browser verification passes

---

## 5. Phase 2 — Communication Workflow Layer

### Entry Gate
- lifecycle stable
- tracking page stable
- staff actions stable
- seeded audit events stable
- communication truth defined

### Exit Gate
- notification event model implemented
- citizen tracker shows communication history correctly
- staff can trigger communication intent actions
- dashboard reflects communication KPIs
- no false sent-state behavior exists
- browser validation proves sync across surfaces

---

## 6. Phase 3 — AI Assistance Layer

### Entry Gate
- Phase 2 exit gate passed
- AI surface contract defined
- advisory boundaries approved
- seeded AI payloads available

### Exit Gate
- AI surfaces are clearly advisory
- AI outputs feel integrated and useful
- staff override remains obvious
- no confusion exists between AI suggestion and human decision
- browser/runtime validation passes

---

## 7. Phase 4 — Administrative Intelligence Layer

### Entry Gate
- Phase 3 exit gate passed
- historical seeded truth exists
- taxonomy and tags approved
- aging and trend formulas defined

### Exit Gate
- analytics trace back to record truth
- trend views are understandable
- dashboard remains coherent
- leadership value is visible
- verification confirms analytic fidelity

---

## 8. Phase 5 — Demo Hardening & Presentation Readiness

### Entry Gate
- targeted features implemented
- major truth conflicts resolved
- artifact backlog under control

### Exit Gate
- golden demo path is stable
- screenshot pack complete
- wording polished
- visual consistency acceptable
- final rehearsal proof captured
- presentation-ready signoff granted
