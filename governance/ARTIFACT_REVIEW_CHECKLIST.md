# Artifact Review Checklist
## CPIMS Proof and Validation Checklist

This checklist governs artifact review for issues, features, and phases.

Artifacts are mandatory proof, not optional extras.

---

## 1. Minimum Artifact Set

For any meaningful feature or phase, collect:

- implementation summary
- screenshots
- acceptance criteria proof
- mismatch report if applicable
- unresolved risks note
- truth touched summary
- follow-up needs

---

## 2. Screenshot Checklist

Check that screenshots show:
- the intended feature in realistic state
- no broken layout
- no contradictory wording
- no internal-only content leaking publicly
- no fake status or fake metrics

---

## 3. Citizen Flow Proof

Where citizen flow is affected, confirm:
- today’s rule is clear
- request path is understandable
- acknowledgement is visible
- tracking reflects correct public-safe state
- communication history appears correctly if applicable

---

## 4. Staff Flow Proof

Where staff flow is affected, confirm:
- triage actions work
- lifecycle actions are valid
- slot assignment works where applicable
- internal notes remain internal
- public notes appear correctly where intended

---

## 5. Dashboard Proof

Where dashboard is affected, confirm:
- numbers match seeded/store truth
- labels match metric definitions
- category/status summaries are accurate
- no fabricated charts or disconnected values exist

---

## 6. AI Surface Proof

Where AI surfaces are affected, confirm:
- AI label is visible
- advisory nature is clear
- output does not appear authoritative
- staff override remains visible
- AI/human distinction is understandable

---

## 7. Communication Proof

Where communication is affected, confirm:
- state transition exists
- channel selection is visible where expected
- citizen-visible history is correct
- dashboard communication metrics are consistent
- no false delivery implication exists

---

## 8. Mismatch Review

Document:
- what does not match truth
- what is visually unclear
- what remains modeled rather than integrated
- what should block merge vs what can be follow-up

---

## 9. Final Review Outcome

Every artifact review must conclude with one of:
- Pass
- Pass with Follow-up
- Blocked
- Needs Truth Clarification
