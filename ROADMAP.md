# Commissioner Public Interaction Management System (CPIMS)
## Execution Roadmap & Agent Control Plan

This roadmap governs CPIMS as a multi-agent, artifact-driven execution program. Antigravity is used as a control plane for disciplined parallel delivery, not as an unconstrained coding assistant. Every new capability must pass through truth definition, controlled lane execution, and browser/runtime verification before it is considered complete.

---

### A. Global Non-Negotiables
1. **Business truth is central**: No lane may redefine day-wise access logic, lifecycle transitions, citizen/public information visibility rules, or metric meanings. (See `TRUTH_REGISTRY.md`).
2. **Advisory AI only**: In all roadmap phases, AI outputs are advisory unless explicitly marked otherwise.
3. **No fake metrics**: All metrics must derive from shared store truth, seeded scenario truth, or modeled event truth.
4. **Verification is mandatory**: No phase is done until Lane 5 validates the golden path in browser/runtime.

---

### B. The 5 Execution Lanes (Agent Boundaries)
1. **Lane A — Product Contract Guardian**: Owns `TRUTH_REGISTRY.md` and business rule fidelity. Protects truth.
2. **Lane B — Citizen Experience Builder**: Owns homepage, submission flows, tracking. Goal: extreme clarity.
3. **Lane C — Staff Operations Builder**: Owns triage, lifecycle actions, slot assignment. Goal: administrative credibility.
4. **Lane D — Executive Visibility Builder**: Owns dashboards, KPIs. Goal: immediate operational value.
5. **Lane E — Verification & Artifacts**: Owns browser validation, golden path testing, screenshots, mismatch reports.

---

### C. Governance Questions Per Phase
Before any phase begins, Lane A (Truth) must answer:
1. What new truth is being introduced?
2. What existing truth is affected?
3. What must remain unchanged for citizen clarity?
4. What is modeled vs actually integrated?
5. What is public-facing vs staff-only vs dashboard-only?
6. What proof will show the phase is complete?

---

### D. Required Artifact Set Per Phase
For each phase, require these outputs:
* feature contract update
* schema/state change summary
* screenshot pack
* golden walkthrough note
* mismatch report
* unresolved risks note
* demo script delta

## The Execution Phases

### Phase 2 — Communication Workflow Layer
**Goal:** Model trustworthy, structured citizen communication without yet depending on real external gateway deployment.

**Entry Gate:**
* lifecycle model stable
* tracking page stable
* staff actions stable
* seeded audit events stable
* dashboard metrics derived from shared truth

**Pass 1 — Truth**
Define: notification event types, channel model, delivery state model, citizen-visible communication history rules, dashboard KPI definitions for communications.

**Pass 2 — Build**
* Citizen Lane: communication history on tracker
* Staff Lane: notify controls and communication intent actions
* Dashboard Lane: communication readiness and delivery-status KPIs

**Pass 3 — Verify**
* staff action creates correct communication event
* citizen tracker reflects it
* dashboard counts update from same truth
* no fake “sent” state without modeled event evidence

**Exit Gate:**
* notification event model implemented
* citizen tracker shows notification history correctly
* staff can trigger notification-intent actions
* dashboard reflects communication KPIs
* browser validation proves end-to-end sync
* Communication history feels real and operationally useful.

---

### Phase 3 — AI Assistance Layer
**Goal:** Model credible AI-assisted capabilities that feel integrated into workflow, not appended for show.

**Entry Gate:**
* Phase 2 exit gate passed.
* TRUTH_REGISTRY AI surfaces defined.

**Pass 1 — Truth**
Define: AI payload contract, advisory/override model, staff visibility rules, citizen visibility rules, confidence / provenance labels if needed.

**Pass 2 — Build**
* Citizen Lane: voice-intake-ready UI and multilingual assistance surfaces
* Staff Lane: AI summary, suggested priority, suggested routing
* Dashboard Lane: AI-usage and AI-assist summary metrics

**Pass 3 — Verify**
* AI surfaces feel organic
* advisory nature is clear
* staff control remains obvious
* no UI confusion between human decision and AI suggestion

**Exit Gate:**
* AI looks like a serious assistive layer, not a decorative layer.

---

### Phase 4 — Administrative Intelligence Layer
**Goal:** Turn operational records into leadership-grade insight.

**Entry Gate:**
* Phase 3 exit gate passed.
* Seeded historical truth available.

**Pass 1 — Truth**
Define: historical data shape, aging formulas, zone/category taxonomy, recurrence logic, duplicate signal model.

**Pass 2 — Build**
* Staff Lane: advanced lookup and historical context
* Dashboard Lane: trend charts, aging buckets, cluster views, zone-wise demand patterns
* Citizen Lane: minimal or no major change

**Pass 3 — Verify**
* all analytics trace back to data truth
* dashboard remains understandable
* leadership insights are immediate, not overcomplicated

**Exit Gate:**
* The dashboard moves from status visibility to decision-support value.
